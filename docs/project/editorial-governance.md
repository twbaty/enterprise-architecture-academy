# Editorial Governance

## Open contribution, controlled acceptance

Anyone may propose material. The repository owner retains final editorial authority over accepted curriculum.

## Required controls

The `main` branch should require pull requests, one approving review, Code Owner approval, the MkDocs build status check, resolved conversations, stale-approval dismissal, and blocked force pushes and deletion.

## Merge authority

Approval and merge are separate decisions. The owner controls both acceptance and release timing.

# Roadmap

This roadmap controls the order of work. Visual polish is secondary to governance, traceability, curriculum completeness, and instructional quality.

## Current state

The repository, MkDocs site, GitHub Actions validation, and GitHub Pages publication pipeline are operational.

The Foundation curriculum outline and initial F1–F4 lesson material exist in working documents but have not yet been fully migrated into canonical Markdown.

**Objective:** Establish a stable source, review, and publication system.

- [x] Create GitHub repository.
- [x] Configure MkDocs Material.
- [x] Configure strict pull-request validation.
- [x] Configure GitHub Pages deployment.
- [x] Establish open contribution with owner approval.
- [x] Consolidate project documentation under `docs/project/`.
- [x] Merge duplicate README content into one root README.
- [x] Establish project philosophy.
- [x] Establish project governance.
- [x] Establish content and style rules.
- [x] Establish ADR structure.
- [ ] Configure the `main` branch ruleset.
- [ ] Confirm `CODEOWNERS` identifies `@twbaty`.

**Exit criterion:** A contributor can propose a change, automated checks can validate it, the owner can approve it, and an accepted merge publishes successfully.

## v0.3 — Foundation curriculum architecture

**Objective:** Define the complete Foundation learning path before filling every chapter.

- [ ] Confirm lesson and chapter sequence.
- [ ] Map each chapter to current official Foundation syllabus objectives.
- [ ] Define prerequisite relationships.
- [ ] Define required lesson components.
- [ ] Define learner, instructor, exercise, and assessment outputs.
- [ ] Define question taxonomy and difficulty scale.
- [ ] Define terminology and glossary ownership.
- [ ] Define chapter completion criteria.
- [ ] Create a curriculum traceability matrix.

**Exit criterion:** Every Foundation objective has an assigned instructional location, exercise strategy, and assessment strategy.

## v0.7 — Practitioner core content

**Objective:** Complete Practitioner lessons, workshops, and scenarios.

- [ ] Complete all Practitioner chapters.

## v0.8 — Integrated release preparation

**Objective:** Validate the complete Academy as one learning system.

- [ ] Cross-check Foundation and Practitioner terminology.
- [ ] Eliminate duplication and contradictions.
- [ ] Validate all internal links and navigation.
- [ ] Validate traceability matrices.
- [ ] Pilot the material with representative learners.
- [ ] Record defects and revise content.
- [ ] Produce release notes and known limitations.

**Exit criterion:** The integrated curriculum has been reviewed and tested as a complete learning path.

## v1.0 — First complete public release

**Objective:** Publish a stable, complete Foundation and Practitioner curriculum.

Required release contents:

- Foundation curriculum;
- Practitioner curriculum;
- persistent case studies;
- learner exercises;
- instructor guidance;
- question banks;
- traceability matrices;
- glossary;
- source and revision records;
- downloadable release artifacts.

Potential future tracks include:

- Healthcare Enterprise Architecture;

These tracks do not begin until the core Foundation and Practitioner path is stable unless the owner explicitly changes the roadmap.

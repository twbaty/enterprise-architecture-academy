# Contributing

Contributions are welcome through GitHub issues and pull requests.

## Editorial authority

The repository owner is the final approving editor. A contribution is not part of the curriculum until it has been reviewed and merged into `main`.

## Recommended contribution process

1. Open an issue describing the correction, lesson, example, or enhancement.
2. Fork the repository.
3. Create a focused branch.
4. Make the change in Markdown.
5. Run `mkdocs build --strict`.
6. Open a pull request using the repository template.
7. Address review comments.
8. The owner approves and merges accepted work.

## Content requirements

Contributions must support the program charter, use authoritative sources, use original wording, contain no recalled exam questions, contain no confidential information, and pass the automated MkDocs build.
